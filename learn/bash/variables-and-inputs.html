Strings, Numbers, and User Input in Bash

In Bash scripting, a string is text, a number is used for calculations, and variables store them for later use. You can also take user input to make your script interactive.

1. Strings

A string is any text inside quotes. 

name="Liam"
echo "My name is $name"

2. Numbers

Numbers donâ€™t need quotes and are used for calculations. 

age=25
echo "I am $age years old"

3. Taking User Input

You can ask the user to type something during script execution. 

echo "Enter your name:"
read username
echo "Hello, $username!"

Explanation:

  • echo displays a message
  • read waits for input and stores it in a variable
  • $username is replaced with the value entered by the user

4. Using read -p

-p lets you ask the question and take input in the same line. 

read -p "Enter your favorite number: " favnum
echo "Your favorite number is $favnum"

Why use read -p?

  • Shorter code
  • Question and input happen in one step

5. Numbers and Calculations

read -p "Enter first number: " num1
read -p "Enter second number: " num2

sum=$((num1 + num2))
echo "The sum is: $sum"

Explanation:

  • $((...)) is for arithmetic in Bash
  • sum stores the result

6. Full Example Script

#!/bin/bash
# This script shows strings, numbers, and user input

read -p "Enter your name: " name
echo "Hello, $name!"

read -p "Enter your age: " age
echo "You are $age years old."

read -p "Enter first number: " n1
read -p "Enter second number: " n2
sum=$((n1 + n2))
echo "The sum of $n1 and $n2 is $sum."

Conclusion

Strings store text, numbers are for math, and variables hold both. With read and read -p, you can easily take input from the user. These basics are essential for writing interactive Bash scripts.
