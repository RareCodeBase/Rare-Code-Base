Cracking Wi-Fi Passwords Using Aircrack-ng

After successfully capturing a WPA/WPA2 handshake, the next step is to crack the Wi-Fi password using a dictionary attack. This guide will walk you through the process using the Aircrack-ng tool.

Legal Notice

Only test networks you own or have explicit permission to audit. Cracking Wi-Fi passwords without consent is illegal and unethical.

Requirements

  • Linux system (e.g., Kali Linux)
  • Captured handshake file (e.g., capture.cap)
  • A wordlist (e.g., /usr/share/wordlists/rockyou.txt)
  • Aircrack-ng installed

Step-by-Step Guide

1. Verify Handshake File

aircrack-ng capture.cap

This checks if the handshake is present. If valid, you’ll see: WPA handshake: [BSSID].

2. Start Dictionary Attack

aircrack-ng -w /path/to/wordlist.txt -b [Router-BSSID] capture.cap

Explanation:

  • -w: Path to your wordlist
  • -b: Target router's BSSID
  • capture.cap: The handshake file

Aircrack-ng will test each password from the list. Duration depends on the list size and complexity.

3. If Password is Found

If the password exists in the list, you’ll see:
KEY FOUND! [ password ]
This is the Wi-Fi password.

4. If Password is Not Found

If not found, try a different or more targeted wordlist. Custom lists based on user info often help.

Example Using rockyou.txt

aircrack-ng -w /usr/share/wordlists/rockyou.txt -b 00:11:22:33:44:55 capture.cap

Tips for Better Results

  • Use leaked or custom wordlists for realistic results
  • Use hashcat for GPU-accelerated cracking
  • Combine lists: cat list1.txt list2.txt > combined.txt

Stopping the Process

Press Ctrl + C anytime to stop the cracking process.

Conclusion

Cracking Wi-Fi passwords from handshake files can be useful for testing security, but it only works if the password is weak or guessable. Always use strong, unique passwords to secure your network.
