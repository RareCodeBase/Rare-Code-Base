If/Else & Conditionals in Bash Scripting

In Bash scripting, if/else statements are used to make decisions. They allow your script to perform different actions based on different conditions.

1. Basic if Statement

#!/bin/bash

num=10

if [ $num -gt 5 ]; then
  echo "Number is greater than 5"
fi

Explanation:

  • if [ CONDITION ] checks if something is true
  • -gt means “greater than”
  • then starts the block of code to run if condition is true
  • fi ends the if statement (it's "if" spelled backward)

2. Using if...else

#!/bin/bash

read -p "Enter a number: " num

if [ $num -ge 18 ]; then
  echo "You are eligible to vote."
else
  echo "You are not eligible to vote."
fi

Explanation:

  • read -p lets you take input from the user
  • -ge means “greater than or equal to”
  • If the number is 18 or more → it prints "eligible"
  • Else → it prints "not eligible"

3. Using elif (else if)

#!/bin/bash

read -p "Enter your marks: " marks

if [ $marks -ge 90 ]; then
  echo "Grade: A"
elif [ $marks -ge 75 ]; then
  echo "Grade: B"
elif [ $marks -ge 50 ]; then
  echo "Grade: C"
else
  echo "Grade: F (Fail)"
fi

Explanation:

  • elif allows checking multiple conditions
  • The script checks each condition one by one from top to bottom
  • Only the first true condition will run

4. Comparison Operators

Operator Meaning

-ne Not equal to
-gt Greater than

-ge Greater than or equal to
-le Less than or equal to

5. String Comparison Example

#!/bin/bash

read -p "Enter your name: " name

if [ "$name" = "admin" ]; then
  echo "Welcome, admin!"
else
  echo "Hello, $name!"
fi

Explanation:

  • "$name" is quoted to avoid errors with spaces
  • = checks if strings are equal

6. Nested If Example

#!/bin/bash

read -p "Enter a number: " n

if [ $n -gt 0 ]; then
  if [ $((n % 2)) -eq 0 ]; then
    echo "Positive and Even"
  else
    echo "Positive and Odd"
  fi
else
  echo "Number is zero or negative"
fi

Explanation:

  • This example uses nested if statements
  • $((n % 2)) is used to check even/odd

7. Tips for Beginners

  • Always use spaces inside [ ]
  • Use double quotes around variables: "$var"
  • Use fi to close every if
  • Use proper indentation to make code easy to read

Conclusion

If/Else conditions are very useful in Bash scripts. They help you make decisions based on values like numbers, strings, or file states. Once you learn how to use them, your scripts become more powerful and interactive.
